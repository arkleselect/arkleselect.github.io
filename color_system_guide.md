# 全站配色系统整合方案 (MiniLoad System Design)

基于对当前代码库及核心组件（HUD、像素风、极简主义）的分析，本项目目前的配色体系主要采用 **高对比度的黑白灰** 搭配 **战术蓝/绿** 点缀。

---

## 1. 核心色彩体系 (Core Palettes)

我们将配色方案分为三个维度：背景层 (Base)、表面层 (Surface) 和交互层 (Interaction)。

| 维度 | 建议颜色 (Hex/OKLCH) | 描述 |
| :--- | :--- | :--- |
| **Main Background** | `#0a0a0c` / `oklch(0.14 0.01 270)` | 极深灰蓝色调，比纯黑更有深度，适合 IPS/OLED 屏幕。 |
| **Card Surface** | `#141417` / `oklch(0.18 0.01 270)` | 略浅的表面色，用于卡片或悬浮层。 |
| **Accent (Primary)** | `#3b82f6` (Blue-500) | 战术科技蓝，用于关键状态、高亮或重要链接。 |
| **Accent (Alternative)** | `#10b981` (Emerald) | 状态在线点、成功提示或特定 HUD 元素。 |
| **Border / Line** | `rgba(255, 255, 255, 0.06)` | 细微的边框，用于勾勒 HUD 轮廓，不产生视觉负担。 |

---

## 2. 字体与排版系统 (Typography)

| 场景 | 字体 | 特性 |
| :--- | :--- | :--- |
| **正文 & 标题** | `Overpass` (Sans/Mono) | 极富现代感的几何字体，具有极高的可读性。 |
| **装饰 & 标识** | `Press Start 2P` | 像素风格，用于系统 ID、状态栏等点睛笔触。 |
| **代码 & 元数据** | `JetBrains Mono` | 极佳的阅读反馈，用于模拟后台数据流。 |

---

## 3. 设计语言整合 (Design Logic)

- **HUD Tactical Layout**: 强调“数据感”，利用细长边框、角落装饰 (Corner Accents) 和呼吸闪烁点 (Pulsating Dots) 营造系统感。
- **Minimalist Content**: 博文区域保持极简，减少色彩干扰，突出文字本身的对比度。
- **Interactive Layers**: 悬浮时边框色由 0.06 提升至 0.2，并引入微弱的蓝色/白色的发光效果。

---

## 4. 配色对比方案对比 (Theme Comparison)

| 元素 | 现状 (Pure Black) | 建议 (Refined Dark) | 优化点 |
| :--- | :--- | :--- | :--- |
| 背景 | `#000000` | `#0a0a0c` (深灰蓝) | 减少视觉压抑感，提升空间深度。 |
| 卡片 | `bg-white/5` | `bg-[#141417]` | 固定色值配合微透明，背景适应性更强。 |
| 文字 | `white/80` | `#e2e8f0` (云母色) | 柔和的白色，降低长时间阅读疲劳。 |
| 边界 | `white/10` | `rgba(255, 255, 255, 0.06)` | 弱化不必要的硬边框，强调内容。 |

---

## 5. 组件级建议

- **Header**: 保持 `backdrop-blur` 的磨砂玻璃效果，背景色可根据全局背景相应加深。
- **Footprints (足迹)**: 维持目前的透明度灰度阶梯，但底色可以更贴合新的深色背景。
- **Daily (日常)**: 时间轴圆点不仅是白色，可以根据状态（如“今天”）切换为 `Accent Blue`。
